Adopt the workdir-centric pipeline from vcs-versioning: version discovery now follows an explicit ``env → config → workdir → version`` chain instead of relying on ambient globals and ``parse`` entry points. The ``egg_info`` command writes ``scm_version.json`` and ``scm_file_list.json`` metadata so sdists can infer versions without a VCS checkout. Requires ``vcs-versioning >= 2.0.0.dev0``.
